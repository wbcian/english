# 英文學習效率研究彙整設計

## 目標

更新 `reference/learning-efficiency-research.md`，把既有調查、兩份 Markdown 報告與兩份 HTML 成果整合成單一可信的 SSOT，並重新核對新增來源與主張。

## 讀者與用途

主要讀者是 Cian；用途是決定日常英文學習配置，以及支撐 app roadmap P13。文件應先回答「怎麼安排最有效」，再交代證據強弱與限制。

## 彙整原則

1. 以系統性回顧、統合分析、原始研究與權威機構資料為優先。
2. 分享頁與下載報告只作為線索；關鍵主張必須回到可公開辨識的原始來源。
3. 不跨不同結果變項直接比較 effect size，也不把研究中的介入長度改寫成「最佳單次時長」。
4. 將內容分成「研究支持」「實務推導」「證據不足／不採用」，避免建議與事實混在一起。
5. 保留現有文件的結論先行結構，補上新增研究帶來的修正、可執行週模板與來源稽核表。

## 預期修改

- 保留 spacing、retrieval practice、程度適配輸入與輸出回饋等高信心原則。
- 重新評估 SRS 是不是能稱為跨技能的「效率之王」。
- 補入 Four Strands 作為時間配置框架，但明示它是教學設計原則，不是精準劑量實驗。
- 將固定分鐘數改為可調整的起始模板，不宣稱為科學最佳值。
- 下修或排除晝夜節律硬排程、固定輸入輸出比例、無可靠來源的神經科學延伸與過度具體的 microlearning 效果。
- 保留對 roadmap P13 的直接含義，但讓功能優先級對應證據強度。

## 驗證標準

- 每個高或中信心主張都有可開啟、可辨識的原始或權威來源。
- DOI、作者、年份、研究類型、樣本／研究數及 effect size 不互相矛盾。
- 文件內沒有 ChatGPT/Gemini 內部 citation token 或無法追溯的引用。
- Markdown 結構、連結與 repo 內引用一致，且 diff 只涵蓋本次彙整。
