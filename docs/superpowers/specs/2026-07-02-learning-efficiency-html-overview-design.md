## 正式總覽：單一路徑報告

採用使用者選定的 A 方案，以由上而下的閱讀流程取代儀表板式卡片堆疊：

1. **答案先行**：spacing、沒有跨技能效率排行榜、microlearning 的邊界。
2. **四個共同原則**：分散、提取、適配、閉環。
3. **證據怎麼改變原結論**：SRS 下修、文法上修、影片與 App 效果量加上研究設計限制。
4. **Cian 的學習策略**：依 TOEIC 750+、成年中高階自學者、現有 Coach Max／App 流程給出優先級。
5. **下週可執行模板**：忙碌日、一般日、每週兩個完整時段，以及每週 Four Strands 偏食檢查。
6. **六技能定位**：閱讀、視聽、單字、文法、口說、寫作的角色、適合碎片化程度與完成條件。
7. **研究限制與來源**：清楚區分統合分析、課程框架與實務推導，連回 SSOT 與 DOI。

### 個人策略原則

- 優先維持每日 spacing／retrieval，但不讓 SRS 吃掉輸入與輸出。
- TOEIC 目標以閱讀／聽力為主要輸入；每次輸入至少留下主旨重述或可重用語塊。
- 文法改為由錯誤與 TOEIC 弱點觸發的短閉環，不安排長篇規則課。
- 每週保留一次完整口說與一次寫作或長篇理解時段。
- 追蹤延後表現與回饋後第二次表現，不以 streak 或完成分鐘數當主要成效。

## 臨時三模型比較

### 模型與資料映射

- **Claude**：原 SSOT 的 deep-research harness 與對抗式查證結果。
- **ChatGPT**：`deep-research-report.md` 與 `preview.html`。
- **Gemini**：`高效英文學習方法研究.md` 與 `code_artifact.html`。

### 閱讀結構

1. 三模型各自的研究風格與來源可稽核性。
2. 共同集合：spacing、主動處理、輸出不可缺、microlearning 有邊界。
3. 獨有增量：Claude 的反證、ChatGPT 的新統合分析與技能完整性、Gemini 的機制與視覺化線索。
4. 分歧：文法配置、SRS 地位、精確分鐘數、Four Strands、晝夜節律。
5. 錯誤／過度推論：無來源效果量、內部 citation token、把框架當最佳比例、把神經機制直接變成處方、把 MALL 等同 microlearning。
6. 最終可保留資訊：每個模型對最終 SSOT 實際貢獻了什麼。

比較頁不做「哪個模型最好」的總排名；評估單位是具體主張及其可追溯來源。

## 視覺與互動

- 正式總覽：暖灰紙張底、深綠主色、琥珀色 caveat；單欄閱讀，少量全寬策略卡與表格。
- 臨時比較：三模型固定色彩，使用交集帶、三欄比較與錯誤警示區。
- 響應式：桌面最大內容寬度約 1100px；手機改為單欄，表格可橫向捲動。
- 無必要 JavaScript；只保留頁內導覽與列印樣式。
- 所有顏色資訊都有文字標籤，不只依賴色彩表達證據強弱。

## 來源與資料流

- 正式總覽只從已複查的 SSOT 取內容，不重新引入下載報告中的未驗證數字。
- 模型比較可引用五份原始 artifact 來描述模型輸出，但判斷真偽時以 SSOT 已核對的 DOI／出版頁為準。
- 分享連結只作 provenance，不作證據。
